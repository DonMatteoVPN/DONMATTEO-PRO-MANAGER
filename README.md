## 🤔 Что это такое?

**DONMATTEO PRO MANAGER** — это программа-меню для управления безопасностью Linux-сервера.  
Вместо того чтобы вводить сложные команды вручную, ты просто выбираешь пункт из меню.

**Что умеет менеджер:**
- 🔒 Настраивать файервол (UFW) — «охрану» на входе сервера
- 🔐 Устанавливать Fail2Ban — автоматически блокирует взломщиков
- 🛡️ Запускать TrafficGuard — защищает VPN от слежки провайдера
- 🔑 Управлять SSH-ключами и портами подключения
- 🧹 Чистить диск от мусора (логи, кэш, Docker образы)
- 💾 Настраивать память (ZRAM + Swap) для экономии RAM
- 🔍 Искать лучшие сервера для VPN-маскировки (Reality TLS Scanner)

## ⚙️ Что делает каждый пункт меню?

### 🔒 1. Файервол (UFW)
**UFW** = Uncomplicated Firewall = «простой файервол»  
Это программа-охранник. Она разрешает только нужные подключения и блокирует всё остальное.

- **Открыть порт** — разрешить подключения на определённый порт
- **Закрыть порт** — заблокировать порт
- **Anti-DDoS** — защита от «залпа» запросов с одного IP
  - CONNLIMIT = лимит одновременных соединений
  - RATELIMIT = лимит новых подключений в минуту

### 🔐 2. Fail2Ban
**Fail2Ban** = «попался — заблокирован»  
Смотрит в логи и блокирует IP-адреса, с которых приходит слишком много неудачных подключений.

- **MaxRetry** = сколько ошибок допустимо (по умолчанию 3)
- **FindTime** = за какое время считаем ошибки (по умолчанию 600 сек = 10 минут)
- **BanTime** = на сколько блокировать (по умолчанию 86400 сек = 24 часа)
- **Whitelist** = список IP, которые НИКОГДА не блокируются (например твой домашний IP)

### 🛡️ 3. TrafficGuard
Автоматически блокирует серверы-«нюхачи» которые пытаются определить,  
что ты используешь VPN через анализ трафика (ТСПУ РКН).

### 🔑 4. SSH-порты
SSH = способ удалённого подключения к серверу (через терминал).  
По умолчанию работает на порту 22 — взломщики об этом знают и атакуют.  
Менеджер помогает добавить нестандартные порты (например 2275, 48912).

> ⚠️ **ВАЖНО:** Перед удалением порта убедись, что можешь зайти через другой!

### 🧹 5. Очистка диска
Удаляет ненужные файлы: старые логи, кэш apt, временные файлы Docker.  
Нормально освобождает 1-10 ГБ на загруженном сервере.

### 💾 6. Память (ZRAM + Swap)
**ZRAM** = сжатая виртуальная память в RAM (быстрее стандартного swap!)  
**Swap** = файл подкачки на диске (медленнее, но больше)  
Вместе (гибридный режим) — лучший вариант для VPS с малым объёмом RAM.

### 🔍 8. TLS Scanner
Ищет серверы с подходящими TLS-сертификатами для маскировки VPN (Reality/XTLS).  
Помогает найти идеальный `ServerName` для твоей конфигурации.

### 🚀 9. Золотой Стандарт 2026
Устанавливает **полный пакет защиты** одним нажатием:
- UFW с оптимальными правилами
- Fail2Ban с настройками для Nginx и SSH
- Оптимизация ядра (BBR, tcp_syncookies, защита от spoofing)
- Гибридная память ZRAM + Swap

### ⚙️ C. Настройки менеджера
Меняй **пути к папкам**, **порты**, **URL источников** прямо из меню — без редактирования кода!

## 🔧 Как добавить свой модуль?

```bash
# 1. Создай файл в папке modules/ (не в core/)
nano /opt/remnawave/DONMATTEO-PRO-MANAGER/modules/m_mymodule.sh

# 2. Начни файл со стандартного заголовка:
#!/bin/bash
# Мой кастомный модуль — описание

menu_mymodule() {
    while true; do
        clear
        ui_header "🔧" "МОЙ МОДУЛЬ"
        echo -e " ${GREEN}1.${NC} Действие первое"
        echo -e " ${CYAN}0.${NC} Назад"
        read -rp ">> " ch < /dev/tty
        case "$ch" in
            1) echo "Делаем что-то!"; ui_pause ;;
            0) return ;;
        esac
    done
}

# 3. Перезапусти don — модуль найдётся автоматически!
don
```

## ❓ FAQ — Частые вопросы

**Q: Почему нужен root?**  
A: Файервол UFW, Fail2Ban, SSH — это системные программы. Изменить их настройки может только администратор (root).

**Q: Что если GitHub заблокирован?**  
A: Менеджер автоматически подбирает рабочее зеркало из списка 7 провайдеров. Работает в России и Китае.

**Q: Я случайно закрыл SSH порт — что делать?**  
A: Подключись к серверу через консоль хостинга (VNC/KVM) и выполни: `ufw allow 22/tcp`

**Q: Почему Fail2Ban не работает после установки?**  
A: Убедись что rsyslog запущен: `systemctl status rsyslog`. Если нет: `apt install rsyslog && systemctl start rsyslog`

**Q: Как посмотреть что делает менеджер с моей системой?**  
A: Аудит-лог: `cat /var/log/don_audit.log`

**Q: Безопасно ли запускать менеджер?**  
A: Да. Все скачиваемые файлы проверяются по SHA256 хешам (мягкая проверка). Исходный код открыт на GitHub.
